# Программа: от идеи до работающего MVP

## Что мы делаем и в каком порядке

| Фаза | Задания | Что делаем | Результат |
|------|---------|-----------|-----------|
| 1. Знакомство с Claude Code | 1-4 | Учимся работать с файлами через AI | Тренировочные файлы в `results/` |
| 2. Продуктовые требования | 5-6 | Описываем продукт через PRD-интервью | `results/PRD.md` |
| 3. Лендинг (маркетинг) | 7-10 | Делаем рекламную страницу для привлечения пользователей | `results/landing-v*.html` |
| 4. MVP (сам продукт) | 11-15 | Пишем и деплоим работающее приложение | Код в `product-mvp/`, приложение на сервере |

> **Лендинг и MVP — это два разных проекта.**
> Лендинг — одностраничный сайт, который РАССКАЗЫВАЕТ о продукте (маркетинг).
> MVP — работающее приложение, которым пользователь ПОЛЬЗУЕТСЯ (сам продукт).
> Мы делаем их по очереди, но это разные вещи с разным результатом.

## Задание 0: Настраиваем Claude Code, чтобы не спрашивал разрешение на каждом шаге (2 минуты)

**Модель:** Sonnet 4.5

**Зачем вам это уметь:** По умолчанию Claude Code спрашивает разрешение на каждое действие — "можно запустить?", "можно прочитать файл?". Это полезно для безопасности, но на курсе будет сильно тормозить. Сейчас мы разрешим ему базовые операции, чтобы он не дёргал вас каждые 10 секунд.

**Введите в Claude Code:**
```
Добавь в файл настроек Claude Code ~/.claude/settings.json разрешения на следующие действия, чтобы ты не спрашивал у меня подтверждение каждый раз:

Инструменты:
- Read (чтение файлов)
- WebSearch (поиск в интернете)
- WebFetch (загрузка веб-страниц)

Команды терминала:
- npm, npx, yarn, pnpm (всё что начинается с этих команд)
- node
- open
- ls, mkdir, cp, mv
- ssh, scp, rsync
- curl
- chmod
- lsof
```

**Результат:**
- Claude Code обновил файл настроек
- Теперь он не будет спрашивать разрешение на чтение файлов, запуск npm и другие базовые команды

**Что по-прежнему будет спрашивать** (и это правильно):
- Создание и редактирование файлов — чтобы вы видели, что именно он пишет
- Удаление файлов — чтобы случайно не потерять что-то важное

# Воркшоп 2: Пишем и деплоим MVP на сервер

## Фаза 4: MVP — работающий прототип продукта

> **Что это:** MVP (Minimum Viable Product) — это работающее приложение, которым пользователь реально ПОЛЬЗУЕТСЯ.
> В отличие от лендинга, MVP не просто рассказывает о продукте — он ДЕЛАЕТ то, что обещает.
>
> **MVP и лендинг — это разные проекты.**
> Лендинг (фаза 3) — статичная рекламная страница для привлечения пользователей.
> MVP (эта фаза) — работающее приложение с логикой, данными и интерфейсом.
>
> Мы берём продуктовые требования из `PRD.md` и пишем настоящий код приложения.

**Результат фазы:**
- Работающее приложение задеплоено на сервер и доступно по ссылке
- Приложение реализует ключевые фичи из PRD.md
- Продуктом можно пользоваться — основной сценарий работает от начала до конца

---

## Задание 11: Пишем код MVP (20 минут)

**Модель:** Opus 4.6

**Зачем вам это уметь:** Вы превращаете документ (PRD) в работающий код. Это главный навык курса — умение пройти путь от идеи до прототипа за один вечер, а не за месяц.

> Мы начинаем писать ПРОДУКТ — рабочее приложение. Это НЕ лендинг.
> Лендинг уже готов в `results/landing.html`, MVP живёт отдельно — в `product-mvp/`.

**Стек технологий:**

| Компонент | Технология | Зачем |
|-----------|------------|-------|
| Фреймворк | **Next.js** (App Router) | Фронтенд + API в одном проекте |
| Стили | **Tailwind CSS** | Быстрая вёрстка без написания CSS |
| База данных | **PostgreSQL** (на учебном VPS) | Хранение данных продукта |
| Деплой | **Учебный VPS** | Приложение доступно по IP |

> **Про базу данных:** Claude Code сам подключится к вашему учебному VPS, установит и настроит PostgreSQL, создаст таблицы и заполнит тестовыми данными.
>
> **Не пугайтесь стека** — Claude Code сам создаст проект, установит зависимости и настроит всё. Вам не нужно знать эти технологии заранее.

**Сначала создайте папку для MVP-проекта** — это отдельный проект, он НЕ должен лежать в `2-3-MVP/results/` (там лежат файлы лендинга и PRD).

**Введите в Claude Code:**
```
Создай папку product-mvp/ для нового проекта. Это будет работающее приложение (MVP), а НЕ лендинг. Лендинг уже готов отдельно в папке 2-3-MVP/results/. Продуктовые требования лежат в файле 2-3-MVP/results/PRD.md — используй их как основу.

/feature-dev
```

После этого отвечайте на вопросы Claude Code. Если он спросит какой продукт или функциональность делаем — сошлитесь на PRD: `2-3-MVP/results/PRD.md`. Подчеркните, что код приложения должен создаваться в папке `product-mvp/`.

**Результат:** Claude Code проведёт интервью, после чего напишет код приложения и предложит протестировать на dev-сервере.
- Claude Code создал файлы проекта (код приложения)
- Это работающее приложение с логикой, а НЕ статичная HTML-страница
- Приложение реализует ключевую функциональность из PRD.md

**Wow-момент:** Из текста продуктовых требований пишет код
