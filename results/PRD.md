# PRD: Анонимная платформа текстовых ролевых игр

**Версия:** MVP 1.0
**Дата:** март 2026
**Статус:** В разработке

---

## 1. Обзор продукта

Веб-сервис для анонимных текстовых ролевых игр. Пользователи публикуют заявки на игру, находят соигроков и ведут совместные истории в закрытых текстовых диалогах. Ключевая ценность — полная анонимность и свобода формата: можно сыграть одну сцену за вечер или вести историю годами, в фандоме или с оригинальными персонажами, вдвоём или с несколькими игроками.

---

## 2. Целевой сегмент

**Текстовый игрок, которому нужна свобода**

Люди 16–35 лет, которые хотят играть в текстовые RP так, чтобы игровая жизнь не пересекалась с реальной — ни с друзьями, ни с фандомом, ни с работой.

**Core Jobs:**

- Когда хочу быстро отыграть острый момент — войти, сыграть и уйти, не оставив следов
- Когда ищу долгую совместную историю — найти соигрока через заявку и развивать её месяцами
- Когда пишу пост — оформить его красиво, с форматированием и атмосферой
- Когда соигрок ведёт себя токсично — просто уйти, без конфликта и последствий

**Big Job:**
Иметь пространство, где можно быть кем угодно и играть что угодно — коротко или долго, легко или глубоко — и получать именно тот опыт, который нужен прямо сейчас, без социальных обязательств и следов.

---

## 3. Рискованные предположения для валидации через MVP

### Риск 1 — Холодный старт (Score: 25 🥇)

**Предположение:** в первые недели после запуска в ленте будет ≥10–15 активных заявок, на которые можно ответить и начать игру в течение 30 минут.
**Почему критично:** пустая лента = нет ценности = пользователь уходит и не возвращается.
**Метод проверки:** закрытая бета в 1–2 фандомных Telegram-чатах, ручной «засев» заявок перед публичным открытием.

### Риск 2 — Токсичность разрушает сообщество (Score: 16 🥈)

**Предположение:** механизмов жалобы + ручной модерации достаточно, чтобы доля игр, прерванных из-за токсичного поведения, не превышала 15%.
**Почему критично:** токсичная площадка отталкивает именно целевую аудиторию и убивает органический рост через сарафанное радио.
**Метод проверки:** подсчёт жалоб на 100 игр в ходе закрытой беты; интервью с 5–7 игроками после беты.

## 4. In Scope (MVP)

### 4.1 Регистрация и авторизация

- Регистрация: email + пароль, **без подтверждения письма**
- Играть можно сразу после регистрации
- Никнейм по умолчанию — «Игрок»
- Никнейм существует только внутри игрового диалога; за пределами игры никнейм нигде не отображается

### 4.2 Лента заявок

**Просмотр и фильтрация:**

- Основная лента всех активных заявок
- Фильтр по: тегам, тексту заявки, названию заявки
- Фильтр по типу игры (двое / мультиплеер)
- Фильтр по уровню контента

**Поведение заявки при отклике:**


| Тип заявки    | Поведение при отклике                                                                                                              |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| На двоих      | Первый отклик → заявка уходит из ленты, создаётся один диалог. Автор может вернуть вручную.                                        |
| Multiplayer   | Заявка остаётся в ленте до ручного снятия автором. Каждый, кто нажимает «Ответить», попадает в **тот же** общий игровой диалог.    |


### 4.3 Создание заявки

**Обязательные поля (без заполнения нельзя опубликовать):**

- Название заявки
- Обязательные теги (категория / жанр — уточняются при разработке)
- Тип игры: двое / мультиплеер
- Уровень контента: ползунок или набор тегов (например: «без постельных сцен» / «редко» / «часто» / «основа сюжета»)

**Необязательные поля:**

- Дополнительные теги
- Текст заявки (через простой текстовый редактор с базовыми тегами)

**Типы заявок:**

- **Публичная** — уходит в ленту при публикации
- **Приватная** — остаётся в черновиках, никогда не попадает в ленту; автор приглашает конкретного человека по ссылке напрямую из черновика

**Статусы заявки:**

- `Черновик` — не видна никому, кроме автора
- `Активна` — видна в ленте, принимает отклики
- `Неактивна` — снята из ленты, хранится в разделе «Неактивные заявки»

**Инвайт-ссылка:**

- Доступна для любой заявки (активной или приватной)
- Принявший ссылку → сразу попадает в игровой диалог
- Для активной публичной заявки: принятие инвайта снимает заявку из ленты (если не включён режим «открытый сбор»)
- Если инвайт проигнорирован или отклонён: заявка остаётся в том же статусе, в котором была (активна в ленте → остаётся активной; в черновиках → остаётся в черновиках)

### 4.4 Игровой диалог

**Участники:**

- Двое (по умолчанию) или несколько (мультиплеер)
- Никнейм внутри игры: можно задать любой, виден только соигрокам; по умолчанию — «Игрок»
- Никнейм существует только внутри игры и нигде более не используется

**Контент диалога:**

- Переписка сообщениями (пост = сообщение в реальном времени без скрытых метаданных: время написания не отображается, статус онлайн не виден, индикатор печати отсутствует)
- Текстовый редактор с базовыми тегами форматирования (жирный, курсив, и др.)
- Баннер-картинка внутри игры (загружается автором игры)
- Аватар персонажа внутри диалога (загружается каждым участником для себя)

**Редактирование постов:**

- Можно редактировать **только свои** посты
- В активной игре: редактирование доступно в течение ~1 месяца после публикации поста
- В завершённой игре: редактирование доступно, но изменения видны **только редактору** — у других участников копия остаётся неизменной

**Экспорт:**

- Возможность экспортировать игровой диалог (формат уточняется при разработке)

**Выход из игры:**

- Нельзя выйти без выбора причины из предустановленного списка (список уточняется при разработке, примеры: «Нет времени», «Сменились интересы», «Неподходящий партнёр», «Другое»)
- Диалог после выхода перемещается в раздел «Завершённые игры» у вышедшего участника
- У оставшихся участников игра остаётся активной

**Правила жизни игры:**

- Закрытие браузера без явного выхода: ничего не происходит — игра остаётся активной, соигроки не получают никаких уведомлений
- Нет индикатора «онлайн»
- В мультиплеере: игра продолжается пока есть хотя бы двое участников, даже если автор заявки вышел. Автор не может принудительно завершить игру для других — только выйти сам.
- Когда все участники вышли → игра переходит в «Завершённые/неактивные игры»

### 4.5 Разделы «Мои заявки» и «Мои игры»

**Мои заявки:**

- Черновики
- Активные заявки
- Неактивные заявки

**Мои игры:**

- Активные игры (с badge-счётчиком новых событий)
- Завершённые / неактивные игры

### 4.6 Закладки

- Сохранение заявок в закладки — только для себя, автор заявки не видит, кто добавил
- В ленте виден статус заявки, добавленной в закладки (активна / в черновиках)
- Лимит: **50 закладок**
- При достижении лимита: предлагается удалить одну из существующих для добавления новой

### 4.7 Уведомления

- **Email-уведомления** о новом сообщении в активной игре (можно отключить в настройках)
- **Favicon-badge:** зелёная цифра на вкладке браузера при наличии новых непрочитанных сообщений
- **Badge-счётчик** на вкладке «Мои игры» внутри интерфейса

### 4.8 Модерация и безопасность

- Кнопка «Пожаловаться» внутри игрового диалога
- Жалоба уходит модераторам с историей диалога
- Модераторы рассматривают вручную; при нарушении — бан аккаунта
- Личной блокировки пользователей нет; пострадавший может выйти из игры
- Случаи для жалоб: злонамеренное нарушение игры, буллинг, преследование

## 5. Out of Scope (MVP)


| Что                                         | Почему отложено                 |
| ------------------------------------------- | ------------------------------- |
| Анонимная стена                             | Концепция не определена         |
| Push-уведомления (мобильные)                 | После выпуска мобильного приложения |
| Рейтинг и репутация игроков                 | Нужна аудитория для калибровки  |
| Автоматический мэтчинг                      | Лента заявок решает задачу      |
| Монетизация                                 | Не в фокусе MVP                 |
| Мобильное приложение                        | После валидации на вебе         |
| История редактирования постов               | Не нужна пользователям          |
| Лайки на заявки                             | Заменяются закладками           |
| AI-соигрок                                  | Следующая итерация              |
| Рейтинг контента 18+ / верификация возраста | Юридическая проработка позже    |

## 6. Edge Cases


| Сценарий                                                         | Поведение                                                                                                                                             |
| ---------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| Пользователь закрыл браузер без выхода из игры                   | Ничего не происходит. Игра активна, соигроки не уведомлены, онлайн-статус не показывается                                                             |
| Два пользователя одновременно нажали «Ответить»                  | Создаются два отдельных диалога. Автор сам решает, с кем играть, остальным может отказать                                                             |
| Один игрок уходит из мультиплеера                                | Игра продолжается для остальных. Игра завершается только когда вышел последний участник                                                               |
| Редактирование поста в активной игре                             | Доступно в течение ~1 месяца с момента публикации поста                                                                                               |
| Редактирование поста в завершённой игре                          | Доступно, но изменения видны только редактору                                                                                                         |
| Пользователь достиг лимита 50 закладок                           | Показывается предложение удалить одну для добавления новой                                                                                            |
| Инвайт-ссылка проигнорирована / отклонена                        | Заявка остаётся у автора. Он сам решает: вернуть в ленту или оставить в черновиках                                                                    |
| Жалоба на токсичного соигрока                                    | Жалоба → модерация → ручная проверка → бан при нарушении. Личная блокировка недоступна                                                                |
| Заявка снята из ленты в момент, когда второй нажимает «Ответить» | Диалог успевает создаться — отклик побеждает. |

## 7. Принципы проектирования

1. **Анонимность по умолчанию** — никакой информации о пользователе не раскрывается без его желания, включая онлайн-статус
2. **Свобода формата** — продукт не диктует длину, жанр или продолжительность игры
3. **Асинхронность** — игры могут длиться годами, выход из сети не влияет на игровой процесс
4. **Безопасный выход** — нельзя бросить партнёра без объяснения (обязательная причина выхода)
5. **Минимум трения при старте** — регистрация без подтверждения, возможность начать играть за 10 минут
