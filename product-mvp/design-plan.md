# 🎨 UX/UI Audit Report — Апокриф

**Дата:** 2026-03-17
**UX Score: 52/100**

---

## 🔴 Критичные UX-проблемы (блокируют пользователей)

### 1. Нет мобильной адаптации
- **Проблема:** 0 медиа-запросов (`@media`), нет мобильного меню, фильтры не складываются на узких экранах. На телефоне интерфейс будет нечитаемым или сломанным.
- **Best practice:** Mobile-first (WCAG 2.1, Apple HIG). 60%+ трафика — мобильные.
- **Рекомендация:** Добавить responsive breakpoints в Tailwind config, hamburger-меню для навигации, stack-layout для фильтров на mobile.
- **Effort:** L

### 2. Нулевая accessibility
- **Проблема:** 1 `aria`-атрибут на весь проект, 0 тегов `<nav>`, `<main>`. `div onClick` без `role="button"`, без `tabIndex`, без `onKeyDown`. Пользователь с клавиатурой или скринридером не сможет пользоваться сайтом.
- **Best practice:** WCAG 2.1 AA — семантические теги, aria-labels, keyboard navigation.
- **Рекомендация:** Заменить `div onClick` на `<button>`, добавить `<nav>`, `<main>`, `aria-label` на интерактивные элементы, `aria-live` для динамического контента (чат, уведомления).
- **Effort:** M

### 3. Нет предупреждения о несохранённых данных
- **Где:** RequestForm, RichEditor, OocEditor
- **Проблема:** Пользователь может случайно закрыть вкладку или нажать «назад» и потерять длинный текст заявки или сообщения. Нет `beforeunload` обработчика.
- **Best practice:** Baymard Institute — всегда предупреждать о потере несохранённого ввода.
- **Рекомендация:** Добавить `window.addEventListener('beforeunload')` при наличии изменений.
- **Effort:** S

## 🟠 Серьёзные проблемы (сильно ухудшают опыт)

### 4. Загрузка — только текст, нет скелетонов
- **Где:** LibraryClient, FeedClient, GamePage
- **Проблема:** При загрузке показывается курсив «Загрузка...». Контент прыгает при появлении. Нет визуального «каркаса» страницы.
- **Best practice:** Skeleton screens (Material Design).
- **Рекомендация:** Добавить skeleton-компоненты для карточек, списков, чата.
- **Effort:** M

### 5. Тихие ошибки API
- **Где:** множество компонентов
- **Проблема:** `fetch` без обработки ошибок — если сервер вернёт 500, пользователь увидит пустой экран без объяснения. Нет retry-механизма.
- **Best practice:** Все 5 состояний UI (empty, loading, error, success, partial).
- **Рекомендация:** Единый компонент `<ErrorState>` с кнопкой «Повторить». Обрабатывать `!res.ok` во всех fetch-вызовах.
- **Effort:** M

### 6. `window.confirm()` для опасных действий
- **Где:** удаление, выход из игры
- **Проблема:** Нативный `confirm()` — выглядит чужеродно, нельзя стилизовать, на мобильных неудобен.
- **Рекомендация:** Создать `<ConfirmDialog>` компонент.
- **Effort:** S

### 7. `alert()` для ошибок
- **Где:** регистрация, настройки
- **Проблема:** `alert('Ошибка')` — блокирует UI, нет контекста, пугает пользователя.
- **Рекомендация:** Добавить toast-систему (неблокирующие, автоисчезающие уведомления).
- **Effort:** S

### 8. Нет автофокуса в формах
- **Где:** Login, Register, RequestForm
- **Проблема:** Пользователь открывает форму и должен вручную кликнуть в первое поле.
- **Рекомендация:** Добавить `autoFocus` на первый input каждой формы.
- **Effort:** XS

## 🟡 Улучшения

### 9. Empty states без CTA
- **Где:** библиотека, лента заявок, список игр
- **Проблема:** «Ничего не найдено» — и всё. Нет подсказки что делать дальше.
- **Рекомендация:** «Ничего не найдено. Попробуйте изменить фильтры или создать свою заявку →»
- **Effort:** S

### 10. Нет подтверждения успешных действий
- **Где:** создание заявки, отправка отклика
- **Рекомендация:** Toast «Заявка создана!» или «Отклик отправлен!»
- **Effort:** S (после создания toast-системы)

### 11. Кнопки загрузки — только текст «...»
- **Проблема:** При отправке кнопка показывает «...» без disabled-состояния. Можно кликнуть повторно.
- **Рекомендация:** `disabled` + спиннер-иконка при `loading`.
- **Effort:** S

### 12. Контраст вторичного текста пограничный
- **Где:** `text-ink-2`
- **Проблема:** Соотношение контраста ~4.8:1 — формально проходит AA, но близко к границе.
- **Рекомендация:** Сделать чуть темнее для лучшей читаемости.
- **Effort:** XS

- Безопасность — iron-session, sanitize, stop-list, rate-limit
