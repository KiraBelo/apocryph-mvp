## Критические проблемы (17)

### Безопасность (3)

**SEC-1: Rate-limit middleware не подключён — мёртвый код**
Файл: `src/proxy.ts`
Простыми словами: Написали защиту от DDoS, но забыли её включить. Файл `proxy.ts` никуда не подключён — нужен `middleware.ts`.

**SEC-2: Заметки (notes) не проходят sanitize — XSS**
Файл: `src/components/game/NotesTab.tsx:112`
Простыми словами: Содержимое заметок вставляется как HTML без очистки. Хотя заметки личные, если санитизация на сервере пропущена — это дыра.

**SEC-3: Banner URL можно инжектить в CSS**
Файл: `GameDialogClient.tsx:182`
Простыми словами: Если в URL баннера вставить спецсимволы `)`, можно «сломать» CSS-стиль и подменить фон. Нужна дополнительная валидация.

### Тихие ошибки клиента — зависание UI (10)

Главная проблема проекта: **на клиенте fetch-вызовы массово не обрабатывают ошибки.** Если сервер недоступен или вернул ошибку — UI зависает или показывает ложный успех.

**SILENT-1: Кубики зависают навечно**
`useDiceRoller.ts:17` — нет try/catch, кнопка остаётся в состоянии «бросаю» навсегда при ошибке сети.

**SILENT-2: Редактирование поста — тихий отказ**
`useGameChat.ts:68` — если сервер отклонил редактирование, пользователь не узнает. Правка потеряна.

**SILENT-3: Пагинация — испорченное состояние**
`useGameChat.ts:86` — при ошибке загрузки страницы `setIcMessages(undefined)` ломает весь чат.

**SILENT-4: OOC история — без catch**
`useGameChat.ts:107` — unhandled promise rejection, `oocLoaded` не ставится, возможны бесконечные повторы.

**SILENT-5: Статус заявки — ложь пользователю**
`MyRequestsClient.tsx:52` — оптимистичное обновление без проверки ответа сервера. UI показывает «опубликовано» даже если сервер отказал.

**SILENT-6: Звёздочка/скрытие игры — рассинхрон**
`MyGamesClient.tsx:79-93` — то же: UI обновляется, сервер может не подтвердить.

**SILENT-7: Выход из игры — уход без подтверждения**
`GameDialogClient.tsx:284` — `router.push('/my/games')` вызывается ДО проверки ответа. Пользователь думает что вышел, но не вышел.

**SILENT-8: Жалоба — ложное «Отправлено»**
`GameDialogClient.tsx:291` — пользователь видит «Жалоба отправлена» даже если сервер не принял. Критично для безопасности.

**SILENT-9: Настройки — тихое несохранение**
`SettingsModal.tsx:59` — нет проверки ответа. Никнейм/аватар меняются локально но не сохраняются.

**SILENT-10: Удаление заметки — призрак**
`useGameNotes.ts:73` — заметка исчезает из UI но остаётся на сервере. При перезагрузке — появляется снова.

### Типы (4)

**TYPE-1: `User` тип не содержит role, banned_at, ban_reason**
`src/lib/auth.ts:4` — используется `as unknown as` чтобы обойти. Если БД изменится — компилятор не поймает.

**TYPE-2: `Message.type` это `string` вместо `'ic' | 'ooc' | 'dice'`**
`src/components/game/types.ts:4` — опечатка в типе не будет поймана.

**TYPE-3: null не проверяется после INSERT RETURNING**
`messages/route.ts:187`, `dice/route.ts:32` — если INSERT не вернёт строку, будет краш.

**TYPE-4: API возвращают `unknown[]` без типизации**
`games/route.ts:11` — клиент получает нетипизированные данные.
