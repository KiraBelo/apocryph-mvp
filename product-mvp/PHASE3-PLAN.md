# Фаза 3 — Архитектурные улучшения (план)

Это крупные задачи, каждая требует отдельной сессии работы.

---

## 1. SSE → PostgreSQL LISTEN/NOTIFY (CRIT-1)
**Зачем:** Сейчас уведомления работают только на одном сервере. Нельзя масштабироваться, при деплое все подключения рвутся.

**Что делать:**
- Заменить in-memory Map в `src/lib/sse.ts` на PostgreSQL LISTEN/NOTIFY
- Каждый серверный процесс подписывается на канал `game_{id}` в PostgreSQL
- При новом сообщении — `NOTIFY game_{id}, '{json}'` вместо прямого вызова listeners
- Позволит запускать несколько серверных процессов (pm2 cluster / несколько инстансов)

**Сложность:** L (2-4 часа работы)
**Зависимости:** Нет

---

## 2. Ограничение роста массива сообщений (CRIT-4)
**Зачем:** Когда партнёр пишет — сообщения бесконечно копятся в памяти браузера. За долгую сессию это тормозит.

**Что делать:**
- В `useGameChat`: при получении SSE-сообщения, если пользователь на последней странице — добавить сообщение и обрезать массив до размера страницы (30 IC / 100 OOC)
- Если не на последней странице — показать индикатор «Новые сообщения ↓» и инкрементировать totalPages
- При клике на индикатор — перейти на последнюю страницу

**Сложность:** M (1-2 часа)
**Зависимости:** Нет

---

## 3. Транзакции для настроек игры (IMP-11)
**Зачем:** Если два игрока одновременно меняют настройки, результат может быть непредсказуемым.

**Что делать:**
- Обернуть PATCH `/api/games/[id]` в `withTransaction()`
- Добавить `SELECT ... FOR UPDATE` на строку игры при изменении `ooc_enabled`
- Проверить все другие эндпоинты где читают+пишут без транзакции

**Сложность:** S (30 мин)
**Зависимости:** Нет

---

## 4. Автоскрытие: порог 3+ / только не-участники (IMP-10)
**Зачем:** Сейчас 2 жалобы скрывают игру. Легко злоупотребить.

**Что делать:**
- В `report/route.ts`: поднять порог автоскрытия с 2 до 3
- Добавить проверку: жалобы от участников игры не считаются (или считаются с меньшим весом)
- Рассмотреть: добавить модераторский дашборд для просмотра жалоб

**Сложность:** S (30 мин)
**Зависимости:** Нет

---

## 5. Доступность: ARIA-атрибуты (IMP-16)
**Зачем:** Люди с ограниченными возможностями не могут пользоваться игровым диалогом.

**Что делать:**
- `Modal.tsx`: добавить `role="dialog"`, `aria-modal="true"`, focus trap
- `TopBar.tsx`: `role="tablist"` / `role="tab"` / `aria-selected`
- `SearchPanel.tsx`: `aria-live="polite"` на результаты
- `MessageFeed.tsx`: `aria-live="polite"` для новых сообщений
- Проверить клавиатурную навигацию (Tab, Escape)

**Сложность:** M (1-2 часа)
**Зависимости:** Нет

---

## 6. Health check эндпоинт (MIN-3)
**Зачем:** Nginx и мониторинг не могут понять жив ли бэкенд.

**Что делать:**
- Создать `src/app/api/health/route.ts`
- GET → `SELECT 1` к БД → 200 OK с `{ status: 'ok', db: 'connected' }`
- При ошибке БД → 503 с `{ status: 'error', db: 'disconnected' }`
- Настроить nginx upstream health check или cron curl + alert

**Сложность:** XS (15 мин)
**Зависимости:** Нет
