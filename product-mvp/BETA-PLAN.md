## Must-have к бета-тесту

### 1. Сброс пароля
- Страница «Забыли пароль?» на логин-странице
- Отправка ссылки на email с токеном (срок действия 1 час)
- Страница ввода нового пароля
- **Зависимости:** настроить отправку email (SMTP или сервис типа Resend/Postmark)

### 2. Email-уведомления
- Новый пост партнёра в игре
- Отклик на вашу заявку
- Предложение завершить/опубликовать игру
- Настраиваемая частота: сразу / раз в час / раз в день / выключить
- **Зависимости:** тот же email-сервис что и для сброса пароля

### 3. Telegram-бот уведомлений
- Привязка Telegram аккаунта к профилю (через бот-токен)
- Те же уведомления что и email: новый пост, отклик, предложение
- Кнопки вкл/выкл для каждого типа
- **Зависимости:** Telegram Bot API, MCP-сервер Telegram уже настроен

### 4. PWA + push-уведомления
- manifest.json + service worker
- «Установить приложение» на Android и iPhone
- Push-уведомления (через Web Push API) как альтернатива email
- Индикатор непрочитанных на иконке приложения
- **Зависимости:** VAPID ключи, service worker

### 5. Автосохранение черновика поста
- Сохранение в localStorage каждые 30 секунд
- При открытии редактора — восстановление черновика
- Отдельный черновик для каждой игры (ключ: gameId + userId + тип)
- Очистка после успешной отправки
- **Зависимости:** нет, чисто фронтенд

### 6. Библиотека: буфер публикации + модерация
- При согласии обоих игроков — статус «на модерации» (не «опубликовано»)
- Сообщение: «Ваша игра будет опубликована после проверки модератором (до 24 часов)»
- Модератор видит очередь на публикацию в админке
- Автоматическая проверка стоп-фразами перед публикацией (уже есть стоп-фразы)
- Модератор: одобрить / отклонить с причиной
- При отклонении — оба игрока видят причину
- **Зависимости:** новый статус модерации для публикации

### 7. Библиотека: предпросмотр + редактирование перед публикацией
- Режим «подготовка к публикации» — оба игрока видят как игра будет выглядеть для читателей
- Возможность: отредактировать посты, удалить неудачные, скрыть случайные OOC в IC
- После того как оба сказали «готово» — запускается модерация (п.6)
- **Зависимости:** п.6

### 8. Множественные отклики на заявку
**Вопрос дизайна (нужно решить):**
- Автор получает несколько откликов → каждый = отдельная игра
- Заявка: настройка «максимум откликов» (1 / 2 / 3 / без лимита)
- При достижении лимита — автоматически скрывается из ленты
- Автор в любой момент может снять заявку вручную
- **Зависимости:** изменения в схеме БД (лимит откликов), логика автоскрытия

### 9. i18n: английский язык
- Завершить перевод src/i18n/en.ts (стабы уже есть)
- Проверить все строки интерфейса
- Переключатель языка в настройках (уже есть)
- **Зависимости:** ручная работа над переводом

### 10. Система предупреждений (обдумать)
- Предупреждение → 2-е предупреждение → временный бан → постоянный бан
- Модератор выбирает: предупредить или сразу бан
- Пользователь видит предупреждение при входе
- **Статус:** внести в план, обдумать когда разберёмся с остальным
